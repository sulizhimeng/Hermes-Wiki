## 架构级新增模块

### 1. Multi-Agent Kanban（v0.13）

`plugins/kanban/` + `hermes_cli/kanban*.py` + `tools/kanban_tools.py`（合计 ~10k 行）。

可持久化的**多 Agent 任务看板**，支持跨 Profile 协作。

- **SQLite + WAL** 持久化（`kanban_db.py:61-68`），原子 claim/dispatch，CAS 防并发
- **9 个状态**：triage / todo / scheduled / ready / running / blocked / review / done / archived（`kanban_db.py:97-98`）
- **Worker 生命周期**：心跳、claim TTL 默认 900s、僵尸进程检测（`os.kill(pid, 0)`）、自动 reclaim
- **熔断**：连续失败计数 → 默认 `failure_limit=3` 自动 block
- **幻觉防护**：completion 时扫描 `summary` 中的 `t_*` ID，对不存在的 ID 拒绝完成
- **多看板**：`~/.hermes/kanban.db`（默认）+ `~/.hermes/kanban/boards/<slug>/kanban.db`
- **自动分解**（`kanban_decompose.py`）：辅助 LLM 把 triage 任务拆成带依赖的子任务图
- **Swarm 拓扑**（`kanban_swarm.py:77`）：workers → verifier → synthesizer，自动建图
- **嵌入式 dispatcher**：gateway/run.py 每 60s tick 一次，可禁用走外部 cron
- **Dashboard 集成**：`plugins/kanban/dashboard/plugin_api.py` 提供 REST + 看板视图
- **22+ MCP tools**：`kanban_complete/block/heartbeat/comment/create/list/...`

详见 [[kanban-multi-agent-board]]。

### 2. `/goal` 与 `/subgoal`（Ralph Loop，v0.13/v0.14）

`hermes_cli/goals.py`（762 行）。把"持续努力直到目标完成"做成一等公民。

- **持续推进**：每回合结束后辅助 LLM 评判 done/continue，未完成则自动注入续推 prompt
- **回合预算**：默认 20 轮，超额自动暂停（`DEFAULT_MAX_TURNS=20`）
- **Fail-open**：判官调用失败 → 默认继续，从不假阳性 done
- **/subgoal**：用户中途补充判定标准，必须**全部**满足才算完成（`goals.py:573-586`）
- **状态持久化**：通过 SessionDB `state_meta` 按 `goal:<session_id>` 存
- **不改 system prompt**：续推 prompt 作为普通 user message 注入 → **保护 prompt cache**
- **CLI/Gateway/TUI 三处对齐**：`cli.py:8739-8804`、`gateway/run.py:10621-10676`、`tui_gateway/server.py:3475-3515`
- **Judge 路由**：`auxiliary.goal_judge.{provider,model,max_tokens}`，默认 max_tokens=4096（推理模型需要）

详见 [[goal-and-ralph-loop]]。

## 自我改进 / Curator

### Curator 成熟（v0.12）

详见 [[skills-system-architecture]] 中 Curator 章节的全面重写。要点：

- **统一 `auxiliary.curator.{provider,model,...}`** 配置槽（`agent/curator.py:1557-1620`）
- **Consolidated vs Pruned** 分类（合并到 umbrella vs 纯过期），三步启发式（`agent/curator.py:498-611, 1021-1049`）
- **每跑一次写报告**：`~/.hermes/logs/curator/{TS}/{run.json, REPORT.md}`
- **新子命令**：`status`/`run [--sync|--background] [--dry-run]`/`pause`/`resume`/`pin`/`unpin`/`archive`/`prune [--days N]`/`list-archived`/`restore`/`backup`/`rollback`
- **Curator backup**：跑前自动 tarball 快照 `~/.hermes/skills/.curator_backups/{TS}/`，可 `hermes curator rollback`
- **`hermes curator run --sync`**：前台跑完返回结果，无需 polling
- **Umbrella-Building Prompt**：要 LLM 建 class-level umbrella，禁止"一 bug 一 skill"碎片化
- **Pinned guard**：`set_pinned(skill, True)` → curator 跳过所有自动转换
- **Provenance**：`tools/skill_provenance.py` 区分 foreground / background_review 写入；curator 只动 agent-created
- **`bump_use()`**：skill_view/preload/invoke 调用时增计数（`#17932`），驱动 active/stale/archived 状态机
