# Hermes Agent 更新日志（2026-04-30 ~ 2026-05-05）

99 个 commit（其中 28 个为 AUTHOR_MAP / chore 收录），核心主题：**Provider 全部插件化**（架构级）+ **Kanban 通用诊断引擎 + 幻觉门**（新模块）+ **i18n 静态消息翻译**（新特性）。源码版本 `0.12.0`（pyproject.toml）。

## 架构级变更

### Provider Profiles：33 个 provider 插件化

两个 PR 串联完成：先把所有 provider 收编到 `providers/` 包下的 `ProviderProfile` ABC（`20a4f79`），再把每个 profile 拆成 `plugins/model-providers/<name>/` 自包含插件（`9022804`）。这是 **平台适配器插件化（v2026.4.23）之后的第二步同构改造** —— 模型 provider 现在和消息平台走同一个路径：bundled 插件 + `$HERMES_HOME/plugins/model-providers/` 用户覆盖（last-writer-wins）。

**新模块 `providers/`**（源码验证）：
- `providers/base.py`（6.7KB，171 行）—— `ProviderProfile` dataclass + `OMIT_TEMPERATURE` sentinel
- `providers/__init__.py`（6.8KB，~200 行）—— `register_provider()` / `get_provider_profile()` / `list_providers()` 注册表 + 惰性发现
- `providers/README.md` —— 完整设计文档

**ProviderProfile 声明字段**（`providers/base.py`）：
- 身份：`name`、`api_mode`（默认 `chat_completions`）、`aliases`
- 元数据：`display_name`、`description`、`signup_url`
- Auth/端点：`env_vars`、`base_url`、`models_url`、`auth_type`（`api_key|oauth_device_code|oauth_external|copilot|aws_sdk`）
- 模型目录：`fallback_models`、`hostname`
- Client 级 quirks：`default_headers`
- Request 级 quirks：`fixed_temperature`（含 `OMIT_TEMPERATURE` sentinel，给 Kimi 用）、`default_max_tokens`、`default_aux_model`

**4 个可重写钩子**：
- `get_hostname()` —— 默认从 `base_url` 解析
- `prepare_messages(messages)` —— provider 特定预处理（Qwen 把消息归一化为 list-of-parts，注入 `cache_control`）
- `build_extra_body(**ctx)` —— OpenRouter 的 `provider_preferences`、Gemini 的 `thinking_config`
- `build_api_kwargs_extras(**ctx)` —— 返回 `(extra_body_additions, top_level_kwargs)`，Kimi 把 `reasoning_effort` 放 top-level，Qwen 拆 `enable_thinking`/`thinking_budget`
- `fetch_models(*, api_key)` —— 默认 GET `{models_url or base_url}/models` 带 Bearer auth；Bedrock/Anthropic/OpenRouter 等覆盖

## 新模块：Kanban 通用诊断引擎

### 幻觉门（hallucination gate）

`feat(kanban): hallucination gate + recovery UX for worker-created-card claims`（`de9238d`，#20232）—— `kanban_complete` 增加可选 `created_cards: list[str]` 字段，让 worker 声明它创建了哪些卡。kernel 校验每个 id 存在且属于当前 worker 的 profile，任一 phantom id 都抛 `HallucinatedCardsError` 并落 `completion_blocked_hallucination` 审计事件。`kanban_db.py:HallucinatedCardsError(ValueError)` 把 phantom 列表挂到 `.phantom`。成功完成的 task 还会对 `summary`+`result` 做非阻塞 prose 扫描，发出 `suspected_hallucinated_references` 事件（针对解析不到的 `t_<hex>` 引用）。

**Recovery API**（`kanban_db.py`）：
- `reclaim_task(task_id)` —— operator 立即释放活跃 worker claim，发 `reclaimed` 事件 + `manual: True` payload（区别于只在 claim 过期后才动作的 `release_stale_claims`）

### 通用诊断引擎（`hermes_cli/kanban_diagnostics.py`，649 行）

`feat(kanban): generic diagnostics engine`（`f67063b`，#20332）—— **取代** PR #20232 的 hallucination 专用 `warnings`/`RecoverySection`，扩到 5 种 distress kind 的 stateless 规则引擎：

| 规则函数 | kind | 触发场景 |
|----------|------|----------|
| `_rule_hallucinated_cards` | `hallucinated_cards` | 阻塞门触发：worker 提交了不存在的 created_cards |
| `_rule_prose_phantom_refs` | `prose_phantom_refs` | summary/result 里出现解析不到的 `t_<hex>` |
| `_rule_repeated_failures` | `repeated_failures` | 同一 task 连续失败 3+ 次（统一 spawn/timeout/crash 计数器后） |
| `_rule_repeated_crashes` | `repeated_crashes` | 同一 task 连续 crash 2+ 次（比 repeated_failures 早） |
| `_rule_stuck_in_blocked` | `stuck_in_blocked` | task 卡在 blocked 太久 |

**严重级别**：`SEVERITY_ORDER = ("warning", "error", "critical")`，UI 涂 amber/orange/red。  
**Action 模型**：每个 `Diagnostic` 携带 `DiagnosticAction` 列表（`reclaim` / `reassign` / 等），UI 渲染按钮，CLI 渲染 hint。  
**自动清除**：底层失败模式恢复（completed 干净事件、spawn 成功、unblock）后停止触发，但审计事件链保留。

### 计数器统一（`fix(kanban): unify failure counter`）

`1fc8733`（#20410）—— Dispatcher 的 circuit breaker 之前**只**保护 spawn 端失败（profile missing、workspace mount 错、exec 失败）。worker spawn 成功但随后超时/crash 的 task 重排回 `ready` 时**不增加计数器**，下个 tick 又被 spawn —— 永远不停。修复：legacy `spawn_failures` 列重命名为 `consecutive_failures`，`enforce_max_runtime`、`detect_crashed_workers`、`spawn_failed` 三条路径统一递增同一计数器，successful complete 才重置为 0（`kanban_db.py:631,754`）。

## 新特性：i18n 静态消息翻译

### `display.language` 配置（`agent/i18n.py`）

`feat(i18n): add display.language for static message translation (zh/ja/de/es)`（`7de3c86`，#20231）—— **故意收窄的 thin slice**：只覆盖审批 prompt、少量 gateway 斜杠回复、restart-drain 通知。Agent 生成内容、log、错误 traceback、tool 输出、斜杠命令描述全部保持英文（避免语义漂移）。

**支持语言**：`SUPPORTED_LANGUAGES = ("en", "zh", "ja", "de", "es")` —— 5 种。  
**目录文件**：`locales/{en,zh,ja,de,es}.yaml`（实测 5 个文件，最多 35 行 / `en.yaml`）。  
**别名解析**（`_LANGUAGE_ALIASES`）：`chinese`、`mandarin`、`zh-CN`、`zh-TW`、`zh-Hans/Hant` → `zh`；`jp`、`ja-JP`、`japanese` → `ja`；类似地覆盖 `de` / `es` / `en`。  
**降级**：未知语言 → `DEFAULT_LANGUAGE = "en"`；catalog 缺 key → 英文 fallback；英文也缺 → 直接返回 dotted key（永远不崩）。

**解析顺序**（`get_language()`）：
1. 显式 `lang=` 参数（传入 `t()`）
2. `HERMES_LANGUAGE` 环境变量
3. `display.language`（config.yaml，`@lru_cache(maxsize=1)`）
4. `"en"`（baseline）

**API**：
```python
from agent.i18n import t
t("approval.choose_long")                       # 当前语言
t("gateway.draining", count=3)                  # {count} 格式化
t("approval.choose_long", lang="zh")            # 显式覆盖
reset_language_cache()                          # config 改了之后调
```

### 同 PR 删除：Gateway 陈旧代码自检

同一 commit 还**反转**了 `_detect_stale_code` / `_trigger_stale_code_restart` 机制（之前 #17648/#19740）—— 每条入站消息比对 boot 时 git HEAD 与磁盘当前 SHA，不一致就回复"代码更新了，请重试"并自动重启。长跑 gateway 的 operator 做 ad-hoc git 操作时会被打断，且无法 opt out。`gateway/run.py` 删除 `_STALE_CODE_SENTINELS`、`_GIT_SHA_CACHE_TTL_SECS`、`_read_git_head_sha()`、`_compute_repo_mtime()`、`_detect_stale_code()`、`_trigger_stale_code_restart()`、boot-snapshot 块等。`tests/gateway/test_stale_code_self_check.py` 整体删除（412 行）。`hermes update` 中的 SIGKILL-survivor sweep（同样标 #17648）已经能处理 supervisor-respawn。
