## 自治 Curator（最终形态）

v0.12.0 把 v0.11.0 末尾引入的 Curator 调成了**完整自治后台 agent**：

- **gateway cron ticker 触发**：长跑 gateway 每 168h 自动跑一次（不再依赖 CLI 启动检查），见 `agent/curator.py` 1781 行（74 KB）+ `hermes_cli/curator.py` 598 行
- **`auxiliary.curator` 统一配置**：`hermes model` 里直接挑 curator 用的辅助模型，dashboard 里改
- **Per-run reports**：每轮写 `logs/curator/run.json` + `REPORT.md`
- **Archived 分类**：归档 skill 区分 *consolidated*（合并）vs *pruned*（淘汰），用 model + heuristic 二选一
- **`hermes curator status`**：列出 most-used / least-used skills，按 `bump_use()` 累计调用次数排序
- **Bundled/hub 双过滤护栏**：Background review + curator 都不能写已 bundled / 已 hub-installed 的 skill
- **`skill_manage` pinned guard**：拒绝任何 create/update/archive/delete 走到 pinned skill

### Self-improvement 循环（review fork）大改造

后台 review fork（每轮结束后决定保不保存技能/记忆）此版本系统重写：

| 改进 | 文件 / 行为 |
|------|------------|
| **Class-first prompt** | 改 rubric 评分代替自由文本 "should this update" |
| **Active-update bias** | 优先 update 本轮刚加载过的 skill，支持 `references/` / `templates/` 子文件 |
| **运行时继承** | fork 继承父 session 的 provider / model / 凭证 —— 之前 fork 用错 provider 是个长期 bug |
| **Scoped toolsets** | review fork 只能用 memory + skills toolset（shell/web 全禁） |
| **Clean shutdown** | memory provider 正确退出 |
| **Clean context** | 上一轮的 tool messages 不进 review 摘要 |

### Provider Registry 抽象化

`providers/__init__.py` + `providers/base.py` 让 provider 变成**可插拔**：

```python
from providers import register_provider, ProviderProfile

register_provider(ProviderProfile(
    name="gmi",
    api_mode="chat_completions",
    display_name="GMI Cloud",
    base_url="https://api.gmi-serving.com/v1",
    env_vars=("GMI_API_KEY",),
    fallback_models=(...),
))
```

## Gateway 平台插件化（最终形态）

`gateway/platform_registry.py` 的 `PlatformRegistry` + `PlatformEntry` dataclass 此版本**全面落地**。`PlatformEntry` 字段扩展（基于源码 `platform_registry.py:38-153`）：

- `name` / `label` / `adapter_factory` / `check_fn`
- `validate_config` / `is_connected` / `required_env` / `install_hint`
- `setup_fn`：交互式 setup（对接 `hermes setup`）
- `source`: `"builtin"` / `"plugin"` + `plugin_name`
- `allowed_users_env` / `allow_all_env`：白名单环境变量名

- `cron_deliver_env_var`：cron `deliver=<name>` 投递目标支持
- `standalone_sender_fn`：cron 独立进程时的发送通道（gateway 不在同一进程时也能投递）

## 核心架构

### `hermes -z` 一次性模式

`hermes -z "<prompt>"` —— 非交互、单次 prompt 模式，支持 `--model` / `--provider` / `HERMES_INFERENCE_MODEL` 环境变量。配合 `hermes update --check`（preflight） + opt-in 的 `HERMES_HOME` 预备份。

### 远程 model catalog manifest

OpenRouter + Nous Portal 模型目录**改从远程 manifest 拉取**，新模型上架不再要等 release —— catalog drift 问题彻底解决。

### Native multimodal image routing

图片路由**按 model 的实际 vision capability 走**，而不是按 provider 默认行为。

### Gateway media parity

Telegram / Discord / Slack / Mattermost / Email / Signal 原生**多图发送**；音频 routing 集中、FLAC 支持、Telegram document fallback。

### Configurable prompt cache TTL

`prompt_caching.cache_ttl`：默认 `5m`，可 opt-in `1h`（bursty session 保温更划算）。详见 [[prompt-caching-optimization]]。

### Secret redaction 默认 OFF（注意：v0.13 又翻回 ON）

v0.12 把默认翻成 off，原因是 redaction 偶尔会破坏 patch 输出。**v0.13 又作为 P0 翻回 ON**（见 [[2026-05-07-update]] 安全章节）。
