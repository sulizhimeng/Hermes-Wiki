## 二、`/goal` 持久目标 + Ralph 循环

**文件**：`hermes_cli/goals.py` (722 行)

详见 [[goal-loop-architecture]]。

设计要点（`hermes_cli/goals.py:DEFAULT_MAX_TURNS=20`）：

- **判官循环**：每轮回复结束后调用辅助模型给出 `{"done": bool, "reason": str}`，未完成则注入 `CONTINUATION_PROMPT_TEMPLATE` 续转
- **持久化**：state 落在 SessionDB `state_meta` 表，key `goal:<session_id>`，`/resume` 自动接续
- **fail-OPEN**：判官报错时一律 continue，turn budget 是最后保险（默认 20 turns）
- **判官 parse 失败保护**：连续 3 次 empty / 非 JSON 回复后自动 pause（防止小模型无法遵守判官的 JSON 契约），API 错误不计数
- **新用户消息抢占**：实时消息优先于续转 prompt；之后仍会重新判官，以便用户消息恰好完成目标时正确退出
- **零系统提示扰动**：续转 prompt 作为普通 user message 追加，**不**改写 system prompt 或 toolset，prompt cache 保持有效
- **`/subgoal`**：用户中途追加的额外验收标准（`8f19078c6`），通过 `CONTINUATION_PROMPT_WITH_SUBGOALS_TEMPLATE` 同时呈现给 agent 和判官

CLI / 网关命令均通过 `hermes_cli/commands.py:105`：

```
/goal <text|pause|resume|clear|status>
/subgoal <text|remove N|clear>
```

> 注：v0.13.0 之后曾有一次 revert（`3e7145e0b`，2026-05-11）回滚了一组 `/goal checklist + /subgoal` 改动，最终 `/subgoal` 在 `8f19078c6` 重新落地。当前 main 上是落地版本。

## 八、Post-write delta lint

**文件**：`tools/file_operations.py:_check_lint_delta`（line 1192）

**两层策略**：

1. **语法层**（in-process / shell linter，微秒级）：捕获 corrupt write / mashed quotes / truncated output。命中即 hot path 返回。
2. **Delta 精化**：post-write 有错时与 pre-write 对比，过滤掉 *已存在* 的错误，只把新引入的错误呈现给 agent。

LSP 语义诊断走独立通道（`tools/file_operations._maybe_lsp_diagnostics`），以 `lsp_diagnostics` 字段附加在 `WriteResult` / `PatchResult` 上，与语法错误并行。

覆盖语言：Python、JSON、YAML、TOML。

## 九、会话恢复（Session auto-resume after restart）

`gateway/run.py:_schedule_resume_pending_sessions()` (line 3161)。

机制：

- session 因 gateway 死锁 / `/update` / 源文件 reload 被强杀时，`resume_pending` 标记落在 SessionDB
- gateway 启动时遍历所有 `resume_pending` session，下一条用户消息进来时走 `_is_resume_pending` 分支自动接续
- transcript 与之前的工具循环都保留

`_should_clear_resume_pending_after_turn()` (line 1122) 控制何时清掉标记。

## 十、`no_agent` cron 模式

**文件**：`cron/jobs.py:498`

新增 `no_agent: bool = False` 参数。当 `True` 时：

- 完全跳过 agent，cron 直接 run `script`
- empty stdout = 静默（不打扰主人）
- non-empty stdout = 投递逐字
- `workdir` 仍然生效，`prompt` 仅作 name hint 使用
- `no_agent=True` 没有 `script` 会被拒绝（line 581）

这是从 "agent-based" 调度收紧成 "script watchdog" 的最简形态，避免简单监控任务被绕一个完整的 LLM turn。

## 十一、ACP `/steer` 与 `/queue`

**文件**：`acp_adapter/server.py:1618 _cmd_steer / 1637 _cmd_queue`

- `/steer <guidance>`：向 *正在跑* 的 agent turn 注入 guidance（不打断），下一次 tool call 之后呈现
- `/queue <prompt>`：把 prompt 排到当前 turn 之后跑，不抢占
- session 状态原子持久化 + reasoning metadata 跨重启保留（`#20279`、`#20296`、`#20433`）

CLI / 网关命令同步注册：`hermes_cli/commands.py:101-104`：

```
/queue <prompt>     # 别名 /q
/steer <prompt>
```

Zed / VS Code / JetBrains 通过 ACP 接入即可使用。
