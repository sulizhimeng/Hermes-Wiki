# Hermes Agent 更新日志（2026-04-29 ~ 2026-05-19）

v2026.4.23 → **v2026.5.16**（hermes-agent 0.14.0），约 2418 个 commit。三大主题：

1. **`run_agent.py` 大重构** —— 10+ 个 helper 模块从单体 `run_agent.py` 抽出到 `agent/*.py`，逐步收敛"巨型类"。
2. **插件架构全面扩展** —— Model Providers / Web / Browser / Image-Gen / Video-Gen / Platform 六大子系统全部迁移到插件目录，骨架统一为 *ABC + Registry + Plugins* 三件套。
3. **新模块 Kanban 编排系统** —— 持久化多 Profile 协作看板，Gateway 内嵌调度器 + Orchestrator 自动分解 + 9 状态机。

## 3. 新模块：Kanban 编排系统

`hermes_cli/kanban*.py` + `tools/kanban_tools.py` + `plugins/kanban/dashboard/` —— 持久化多 Profile 协作看板。

**核心**：每个 board 一份 SQLite (`kanban.db`)，9 个状态（`kanban_db.py:97`）`triage → todo → scheduled → ready → running → blocked → review → done → archived`；Gateway 主循环里 `_kanban_dispatcher_watcher` 每 `dispatch_interval_seconds`（默认 60s）调 `dispatch_once`（`kanban_db.py:4595`）跑一次：reclaim stale/crashed → recompute_ready → claim+spawn → review-column dispatch。

**Worker 启动命令**（`_default_spawn`, `kanban_db.py:5160`）：
```
hermes -p <assignee> --accept-hooks ... chat -q "work kanban task <id>"
```
board 通过 `HERMES_KANBAN_*` env 钉死，自动加载 `kanban-worker` skill，按需注入 `--skills` 和 `-m model_override`。

**Orchestrator**：`decompose_triage_task`（`kanban_db.py:3126`）用 aux LLM 把 triage 卡片扇出成子任务图，原子 write_txn + Kahn 环路检测，根任务保留作 parent。门控：`kanban.auto_decompose` / `auto_decompose_per_tick`（默认 3）。

**并发与守卫**：
- `max_spawn` —— 实时并发上限（计 `status='running'`）
- `max_in_progress` —— 达到时整 tick 跳过
- `failure_limit` —— 断路器，默认 2（`DEFAULT_FAILURE_LIMIT`）
- `check_respawn_guard`（`kanban_db.py:4469`）—— 三类阻断：`blocker_auth` / `recent_success`（3600s 窗口）/ `active_pr`（86400s 窗口）

**MCP 工具**（`kanban_tools.py:1218`）：`kanban_show, kanban_list, kanban_create, kanban_complete, kanban_block, kanban_comment, kanban_heartbeat, kanban_link, kanban_unblock`。

**CLI** `hermes kanban`：`init, boards, create, swarm, list, show, assign, reclaim, reassign, diagnostics, link, claim, comment, complete, edit, block, schedule, unblock, archive, tail, runs, heartbeat, log, gc, dispatch, watch, stats, notify-subscribe/list/unsubscribe, specify, decompose`。

## 4. 新机制

### `/goal` —— 跨轮持久目标（Ralph loop）

`hermes_cli/goals.py`（762 行）`GoalManager`。Free-form 用户目标跨轮保活，每轮结束用 aux model 做 judge：达成则结束，否则把 continuation prompt 当普通 user message 追加回同一 session，直至 budget 耗尽 / 用户暂停 / 新消息进来抢占（同时暂停 goal 循环）。

**不变量**：
- continuation prompt 是普通 user message，不动 system prompt / toolset → prompt cache 不失效
- judge 失败 fail-OPEN（继续）—— 坏 judge 不能卡住进度，turn budget 是兜底
- 用户新消息抢占当轮，但仍在 turn 后 re-judge
- 状态进 SessionDB 的 `state_meta` 表 `goal:<session_id>`，`/resume` 可恢复

**配套**：`/subgoal` 把用户后续条件追加进 active goal。

### `/handoff` —— 跨平台会话转移

`hermes_state.py:218` 引入 `handoff_state` / `handoff_platform` / `handoff_error` 字段；`request_handoff()`（`hermes_state.py:3151`）状态机：`None → pending → accepted → completed | failed`。CLI 发起，gateway 拾起后接管该 session。

### `/update` slash 命令

`feat(cli): add /update slash command to CLI and TUI` —— `hermes update` 现在也可以在 CLI/TUI 里通过斜杠命令触发。`feat(update): support pip install --upgrade for PyPI installs`、`feat(update): --yes/-y` 跳过交互。

`feat(update): syntax-validate critical files post-pull, auto-rollback on failure` —— 拉完代码后会语法校验关键文件，失败自动回滚。

### Session Search 单形态

`tools/session_search_tool.py` 改为**无 mode 参数**的单形态工具，三种调用形态由参数推断：

| 形态 | 触发 | 输出 |
|------|------|------|
| **DISCOVERY** | 传 `query` | FTS5 hit + ±5 消息窗 + bookend_start/end |
| **SCROLL** | 传 `session_id` + `around_message_id` | ±window 消息，无 FTS5，无 bookend |
| **BROWSE** | 无参数 | 最近 session 时间序 |

**关键变更**：删除原 LLM summary 路径，所有形态直接读 SQLite，**零 LLM 成本**。详见 [[session-search-and-sessiondb]]（待同步）。

### Checkpoints v2 —— 单存储 + 真剪枝

`tools/checkpoint_manager.py` 重写。原 v1 每个 workdir 一份 shadow git repo；v2 用**单一共享 store**（`~/.hermes/checkpoints/store/`），git objects 在所有项目间去重，per-project 只存 `refs/hermes/<hash16>` + `indexes/<hash16>` + `projects/<hash16>.json`。

特性：
- 真剪枝 + 磁盘 guardrail
- 首次 init 时自动迁移 legacy `legacy-<timestamp>/` 备份
- `GIT_CONFIG_NOSYSTEM=1` belt-and-suspenders
