# Hermes Agent 更新日志（2026-04-30 ~ 2026-05-17）

发布 **v2026.5.16**（pyproject `0.14.0`），988 个 commit。两条主线：**`run_agent.py` 巨型重构**（拆分为 15+ 个 `agent/` 子模块）+ **Provider 全面插件化**（`web` / `browser` 内置 provider 目录删除，迁移为插件 + ABC + 注册表）。此外新增 Kanban 编排器自动分解、xAI Grok OAuth、NovitaAI、`video_generate` 工具、LSP 语义诊断、SimpleX/LINE 平台等。

---

## 架构级变更

### `run_agent.py` 巨型重构

历史上的单体 `run_agent.py` 被系统性拆分。`AIAgent` 仍定义在 `run_agent.py:326`，但其 `__init__`、`run_conversation`、流式 API caller、工具执行、系统提示构建、上下文压缩、后台 review 等均抽到独立模块，`AIAgent` 上的同名方法变成薄转发器。

| 新模块 | 内容 | 规模 |
|--------|------|------|
| `agent/agent_init.py` | `init_agent()`（原 1381 行 `__init__`） | 1469 行 |
| `agent/conversation_loop.py` | `run_conversation()` 主循环、工具派发、重试、压缩触发 | 4018 行 |
| `agent/system_prompt.py` | `build_system_prompt()` —— 系统提示**装配层** | 333 行 |
| `agent/chat_completion_helpers.py` | 流式 chat-completion API caller | — |
| `agent/tool_executor.py` | 工具执行 | — |
| `agent/conversation_compression.py` | 压缩驱动逻辑（`compress_context` 等） | 556 行 |
| `agent/background_review.py` | 后台 memory/skill review、review prompts | — |
| `agent/tool_dispatch_helpers.py` | 工具派发辅助 | — |
| `agent/message_sanitization.py` | 消息净化 | — |

`run_agent.py` 现在约 4096 行（之前为单体）。详见 [[agent-loop-and-prompt-assembly]]、[[aiagent-class]]。

### 系统提示三层装配

`build_system_prompt()`（`agent/system_prompt.py:274`）不再产出固定顺序的 `prompt_parts` 列表，而是经 `build_system_prompt_parts()`（`system_prompt.py:60`）产出**三个命名层**，再用 `"\n\n"` join：

- **stable** —— 身份 / SOUL.md / 工具指导 / skills / 环境 / 平台提示 / 模型指导（构建一次，仅压缩时重建，保护 prefix cache）
- **context** —— 上下文文件 + caller 传入的 `system_message`
- **volatile** —— 记忆快照 / USER.md / 外部 provider / 时间戳行

`agent/prompt_builder.py`（现 1456 行）退化为组件 builder（`build_skills_system_prompt` 等），装配/排序职责转移到 `agent/system_prompt.py`。详见 [[prompt-builder-architecture]]。

### Web / Browser Provider 插件化（重大架构）

`tools/web_providers/` 与 `tools/browser_providers/` 两个内置目录被**删除**，所有 provider 迁移为插件：

**Web Search**：新增 `WebSearchProvider` ABC（`agent/web_search_provider.py:63`，带 `supports_search` / `supports_extract` / `supports_crawl` 能力标志）+ `agent/web_search_registry.py` 注册表。7 个 provider 全部迁为 `plugins/web/` 下插件：`firecrawl` / `exa` / `parallel` / `tavily`（多能力）+ `brave-free` / `ddgs` / `searxng`（仅搜索）。三个 web 工具经 `web_search_registry` 的 `get_active_search_provider()` / `get_active_extract_provider()` / `get_active_crawl_provider()` 派发；新增 per-capability 配置键 `web.search_backend` / `web.extract_backend` / `web.crawl_backend`。

**Browser**：新增 `BrowserProvider` ABC（`agent/browser_provider.py:49`，抽象方法 `name` / `is_available()` / `create_session(task_id)` / `close_session()` / `emergency_cleanup()`）+ `agent/browser_registry.py`。`browserbase` / `browser-use` / `firecrawl` 迁为 `plugins/browser/` 插件。`_get_cloud_provider()` 改为经 `agent.browser_registry` 派发；自动探测顺序 `_LEGACY_PREFERENCE = ("browser-use", "browserbase")`，Firecrawl **被故意排除**自动探测（仅 `browser.cloud_provider: firecrawl` 显式可用）。

详见 [[web-tools-architecture]]、[[browser-tool-architecture]]。

## 多 Agent：Kanban 编排器

Kanban 编排器演进为**跨进程**多 Agent 层（区别于会话内的 delegate/MoA/background-review）。`hermes_cli/kanban*.py`（`kanban.py` / `kanban_db.py` / `kanban_decompose.py` / `kanban_diagnostics.py` / `kanban_specify.py`）+ gateway dispatcher 通过 `sys.executable -m hermes` 派生 worker 子进程。

- **triage 自动分解**（#27572）：新 triage 任务被自动分解为 ready 工作图。配置 `kanban.auto_decompose`（默认 `True`），每 tick 上限 `kanban.auto_decompose_per_tick`（默认 `3`）。`decompose_triage_task`（`kanban_db.py:2780`）按描述把子任务路由到 specialist profile。
- **Step-0 profile discovery**：编排器丢弃硬编码 specialist 名册，改为读取配置的 profile 名册（`_build_roster`，`kanban_decompose.py:214`）。#27572 同时给 `ProfileInfo` 加 `description` / `description_auto`，新增 `hermes profile describe`。
- **`stranded_in_ready` 诊断**（#23578）：`kanban_diagnostics.py:571` `_rule_stranded_in_ready`，阈值 `stranded_threshold_seconds` 默认 `1800`（30 分钟），标记长期未认领的 ready 任务。
- `max_spawn` 是**实时并发上限**，非 per-tick 预算。

详见 [[multi-agent-architecture]]。
