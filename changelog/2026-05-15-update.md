# Hermes Agent 更新日志（2026-04-30 ~ 2026-05-15）

当前版本 **v0.13.0**（最新发布 tag `v2026.4.30`），自上个同步窗口起 579 个 commit。核心主题：**Web 后端插件化**（架构级重构）+ **Codex App-Server 运行时** + **LSP 语义诊断** + **供应链安全加固** + **RL 基础设施移除**。

所有结论均经 `NousResearch/hermes-agent` HEAD（2026-05-15）源码验证。

---

## 架构级变更

### Web 后端插件化（重大重构）

所有 Web 搜索/提取/爬取后端迁移到**插件架构**。旧的 in-tree 目录 `tools/web_providers/` 已删除，每个后端现在是 `plugins/web/<name>/` 下的内置插件（`kind: backend`，含 `plugin.yaml` + `provider.py`）。

- **`WebSearchProvider` ABC**（`agent/web_search_provider.py`）：唯一的插件对接面，镜像 image_gen 模板。`supports_search/extract/crawl` 能力标志 + `search/extract/crawl` 方法（extract/crawl 可为 `async def`）。
- **注册表**（`agent/web_search_registry.py`）：`name → provider` 全局映射，`get_active_{search,extract,crawl}_provider()` 按能力解析活跃后端。
- **插件 facade**：`PluginContext.register_web_search_provider()`。
- **七大后端**（原 4 个）：`firecrawl`、`tavily`（均支持 crawl）、`parallel`、`exa`、`searxng`、`brave-free`、`ddgs`。
- per-capability 配置键 `web.{search,extract,crawl}_backend` 允许不同能力用不同后端。

详见 [[web-tools-architecture]]。

### Codex App-Server 运行时（可选 opt-in，#24182）

为 OpenAI/Codex 模型新增**可选的替代运行时**：把每个回合交给一个 `codex app-server` 子进程，而非走 Hermes 自己的工具派发循环。**默认行为不变。**

- 新模块 `agent/transports/codex_app_server.py`（368 行，stdio 上的 JSON-RPC 2.0）、`codex_app_server_session.py`（810 行）、`codex_event_projector.py`（312 行，把 codex `item/*` 通知投影回 Hermes 标准消息形状）。
- `_VALID_API_MODES` 新增 `codex_app_server`；`_maybe_apply_codex_app_server_runtime()` 仅当 `model.openai_runtime: codex_app_server` 且 provider ∈ `{openai, openai-codex}` 时才改写 api_mode。
- 并非通过标准 `register_transport()` 注册——`AIAgent` 直接分支。

详见 [[provider-transport-architecture]]、[[smart-model-routing]]。

### RL 强化学习基础设施移除（#26106）

commit `5af672c` **彻底移除** Atropos RL 环境与 tinker-atropos 集成：删除整个 `environments/` 目录（43 文件）、`rl_cli.py`、`tools/rl_training_tool.py`（全部 10 个 `rl_*` 工具）、`tinker-atropos` submodule，以及 `pyproject.toml` 的 `rl` / `yc-bench` extras。

数据生成系统**当前只剩 SFT**：轨迹保存（`agent/trajectory.py`）+ 批量运行器（`batch_runner.py`）+ 工具集分布（`toolset_distributions.py`）仍保留。

详见 [[trajectory-and-data-generation]]。

## 工具与能力

### write_file / patch 的 LSP 语义诊断（#24168）

`write_file` 与 `patch` 的写后检查不再只做语法检查，现接入**真实语言服务器**返回语义诊断（类型错误、未定义名、缺失导入）。

- 新模块 `agent/lsp/`（protocol/client/workspace/servers/install/manager/reporter/range_shift/cli/eventlog），约 26 个语言服务器（pyright、gopls、rust-analyzer 等）。
- 分层写后检查：进程内语法检查 → 语法干净时才跑 LSP 语义诊断。
- 仅在检测到 git 工作区时启用；所有 LSP 失败静默回退到纯语法结果。
- 诊断针对写入开始时的基线 delta-filter；`#25978` 把基线诊断重映射到编辑后坐标系。

详见 [[tool-registry-architecture]]。

### 每轮文件变更校验页脚（#24498）

针对模型「发起多个并行 patch、部分失败、却声称已修改全部文件」的过度声称问题，Agent 在每轮结束时追加**文件变更校验页脚**。

- 纯事后检查：只读本轮工具结果，不发起新 LLM 调用、不注入合成历史消息。
- `_turn_failed_file_mutations` 按路径维护状态，first-error-wins，后续成功写入清除失败条目。
- `FILE_MUTATING_TOOL_NAMES`（= `{write_file, patch}`）抽到共享模块 `agent/tool_result_classification.py`。
- 开关 `display.file_mutation_verifier`（默认 true）/ `HERMES_FILE_MUTATION_VERIFIER`。

### video_generate 统一工具（#25126）

新增统一的 `video_generate` 工具，采用可插拔 provider 后端（镜像 image_gen 模式）。

### 其他工具增强

| 增强 | 说明 |
|------|------|
| **GLM 工具使用强制** | `glm` 加入 `TOOL_USE_ENFORCEMENT_MODELS`——GLM 系模型同样有 describe-instead-of-call 故障模式 |
| **线程级工具白名单** | `set_thread_tool_whitelist` / `clear_thread_tool_whitelist`（`hermes_cli/plugins.py`），在 `pre_tool_call` gate 限制可通过的工具，被 `_spawn_background_review` 使用 |
| **HERMES_SESSION_ID** | 通过 ContextVar + 环境变量暴露给 agent 工具（#23847） |

## 安全加固

### 供应链咨询检查器（`hermes_cli/security_advisories.py`，451 行）

针对 PyPI 单包投毒攻击（如命中 `mistralai==2.4.6` 的 Mini Shai-Hulud 蠕虫），新增运行时检查器：`ADVISORIES` 目录 + `detect_compromised()`（用 `importlib.metadata` 检测，不依赖 pip），接入 `hermes doctor` / `cli.py` / `gateway/run.py`。

### 懒安装框架 + 分层安装回退（`tools/lazy_deps.py`，608 行）

opt-in 后端改为**首次使用时按需安装**：`LAZY_DEPS` 白名单 + `ensure()`（`uv → pip → ensurepip` 阶梯）+ 严格 spec 安全正则。被隔离/撤回的 PyPI 包不再静默把安装降级为「仅核心」。配套为 5 个宽松依赖添加版本上界并文档化供应链策略。

### YOLO 模式可见性

`--yolo` 模式（绕过所有审批）现在在 banner（红色一行）与状态栏（红色 `⚠ YOLO` 片段）显式展示。

详见 [[security-defense-system]]。

## Prompt 缓存与上下文压缩

### 长效前缀缓存回退 + 系统提示字节静态不变量

曾短暂引入的跨会话 1h 长效前缀缓存布局（#23828）被 `#24778` **整体回退**——易变层（时间戳/memory/USER profile）每轮变化导致系统消息字节在会话中途变化，反而打穿上游缓存。回归单一 `system_and_3` 布局，并确立不变量：**系统提示在一个会话内字节级静态**。`#24702` 另外对 Portal Qwen 去掉 `ttl=1h`（Alibaba DashScope 上游仅支持 5m）。

### `protect_first_n` 可配置

上下文压缩头部保护从硬编码 3 改为可配置参数 `compression.protect_first_n`（默认 **3**）；新语义为「系统提示之外额外保护的非系统头部消息数」，系统提示始终隐式受保护。`ContextEngine` ABC 默认值同步对齐为 3。

详见 [[prompt-caching-optimization]]、[[context-compressor-architecture]]。
