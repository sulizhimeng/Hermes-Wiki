# Hermes Agent 更新日志（2026-04-30 ~ 2026-05-18）

版本 **v0.14.0**，285 个 commit（非 merge）。核心主题：**`run_agent.py` 大重构**（约 12 个模块抽取到 `agent/` 包）+ **session_search 单形态无 LLM 重写** + **浏览器 Provider 插件化** + **辅助客户端分层 fallback ladder** + 多个新 Provider（xAI Grok OAuth / Azure Foundry / NVIDIA NIM）。

---

## 架构级变更

### `run_agent.py` 大重构（模块抽取）

历史上 3,600+ 行的单体 `run_agent.py` 被拆分：约 12 个辅助模块被抽取到 `agent/` 包，`run_agent.py` 收窄为聚焦 `AIAgent` orchestrator 类（仍 4,115 行，`class AIAgent` 在 `run_agent.py:326`）。AIAgent 上的许多方法现在只是**薄转发器**（thin forwarder），真正实现在 `agent/` 下：

| 抽取模块 | 职责 |
|---------|------|
| `agent/conversation_loop.py` | 对话循环本体（`run_conversation` 在 `:187`，循环在 `:598`） |
| `agent/system_prompt.py` | 系统提示三层组装（`build_system_prompt_parts`） |
| `agent/tool_executor.py` | 工具执行（concurrent / sequential，`handle_function_call`） |
| `agent/chat_completion_helpers.py` | 流式 API 调用、`handle_max_iterations` |
| `agent/agent_init.py` | `__init__`（约 1,381 LOC） |
| `agent/conversation_compression.py` | 上下文压缩编排 |
| `agent/background_review.py` | 后台 memory / skill review |
| `agent/tool_dispatch_helpers.py` | 并发安全判定 |
| `agent/message_sanitization.py` | 消息净化 |
| `agent/stream_diag.py` | 流诊断 |
| `agent/codex_runtime.py` | Codex app-server 运行时 |
| `agent/agent_runtime_helpers.py` | 杂项运行时 helper |

详见 [[agent-loop-and-prompt-assembly]]、[[aiagent-class]]。

### 系统提示三层模型

`build_system_prompt_parts`（`agent/system_prompt.py:60`）不再产出扁平的「13 段 `prompt_parts` 列表」，而是返回含**三个有序层级**的 dict——`stable` / `context` / `volatile`——按内容稳定度从前到后排列，以最大化上游 prefix cache 命中率。时间戳行同时改为**日期精度**（`%A, %B %d, %Y`，commit `4a3f13b`），避免分钟级精度每轮使 KV cache 失效。详见 [[agent-loop-and-prompt-assembly]]。

### 浏览器 Provider 插件化

云端浏览器 Provider 从内置目录 `tools/browser_providers/`（已删除）迁移为插件系统：

- 新 ABC `BrowserProvider`（`agent/browser_provider.py:49`），5 个抽象成员：`name` / `is_available()` / `create_session()` / `close_session()` / `emergency_cleanup()`，可选 `get_setup_schema()`
- 新注册表 `agent/browser_registry.py`——线程安全 `_providers` dict，插件经 `ctx.register_browser_provider(...)` 注册
- 三个内置插件：`plugins/browser/{browser_use,browserbase,firecrawl}/`
- `_get_cloud_provider` 改为经 `agent.browser_registry` 解析；Firecrawl 永不自动探测，仅显式 `browser.cloud_provider: firecrawl` 可达

详见 [[browser-tool-architecture]]。

## session_search 单形态重写（#27590）

`session_search` 工具被重写为**单形态（single-shape）工具，全程无 LLM 调用**——之前的「FTS5 + LLM 摘要」模式中的摘要步骤被彻底移除，每种形态都直接返回数据库里的真实消息。三种模式由参数推断（无显式 `mode` 参数）：

- **DISCOVERY**（`query=`）：FTS5 检索，按 lineage root 去重，每个命中返回 `snippet`、±5 条消息窗口、`bookend_start/end`
- **SCROLL**（`session_id` + `around_message_id`）：以锚点为中心 ±`window` 条消息
- **BROWSE**（无参数）：近期 session 列表

详见 [[session-search-and-sessiondb]]。

## 上下文压缩增强

| 功能 | 说明 |
|------|------|
| **abort-on-summary-failure**（#28102/#28117） | 新构造参数 `abort_on_summary_failure`（默认 `False`）。开启后摘要 LLM 失败时压缩**中止**、消息原样返回，会话冻结至下次 `/compress` 或 `/new`；关闭时沿用旧逻辑（插占位符 + 丢弃中段）。配置项 `compression.abort_on_summary_failure` |
| **strip historical media**（#27189） | `compress()` 末步新增 `_strip_historical_media()`：以最新含图消息为锚，把更早消息里的图片替换为占位符，避免数 MB base64 图块每轮重复上传 |

详见 [[context-compressor-architecture]]。

## 辅助客户端分层 fallback ladder

显式辅助 Provider 失败时新增分层降级链（commit `a574246`）：(1) 主辅助 Provider → (2) 配置 `auxiliary.<task>.fallback_chain` → (3) **主 agent Provider+模型安全网** → (4) 告警并重抛。新函数 `_try_configured_fallback_chain()` / `_try_main_agent_model_fallback()`。`_is_payment_error` 现在也把**每日配额耗尽**视为 payment error（"quota exceeded" / "tokens per day" / "resource exhausted"）。详见 [[auxiliary-client-architecture]]。

## OAuth 凭证健壮性

| 修复 | 说明 |
|------|------|
| **死 token 隔离** | MiniMax / Codex / xAI OAuth 在终结性 refresh 失败时，从 `auth.json` 剥离 `access_token`/`refresh_token`/`expires_at` 并写入 `last_auth_error`，后续调用快速失败、不再网络重试（沿用 Nous 模式） |
| **Nous invoke JWT** | Nous 推理改为优先使用受限作用域的 `invoke_jwt`，失败时回退旧的 24h 不透明 session key |
| **xAI entitlement 分类** | `classify_api_error` 新增最高优先级模式块：xAI Grok entitlement SSE 错误（`status_code=None`）分类为 `auth`、不重试、应 fallback，不再耗尽 `max_retries` |

详见 [[credential-pool-and-isolation]]、[[interrupt-and-fault-tolerance]]。`FailoverReason` 枚举从 13 项增至 19 项。
